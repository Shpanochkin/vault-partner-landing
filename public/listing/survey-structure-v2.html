Что это за документ. Полная спецификация для создания Zoho Survey, который используется на Stage 2 Blueprint-процесса листинга. Каждый вопрос имеет ID, тип поля, обязательность, опции ответа, правила ветвления и маппинг на custom field в Deal. По этой спецификации команда автоматизации собирает Survey в админке Zoho Survey, привязывает webhook и пишет listing_handle_survey_webhook().

1. Обзор

Цель. Собрать с B2B-партнёра всю информацию и документы, необходимые для принятия решения о листинге его токена в Vault, в формате, который позволяет автоматически:

  1. Распознавать стоп-факторы (security / анонимность / санкции / утилити) и падать в terminal-стадии Blueprint-а без ручного разбора.
  2. Определять режим листинга (Full / Partial / Virtual-Only / Custom-Negotiation) по матрице market×network, не заставляя клиента вручную выбирать.
  3. Загружать все обязательные документы (whitepaper, legal opinion, registry extract, audit report) сразу в WorkDrive через upload-поля, без последующих email-overhead.
  4. Мапить ответы 1-в-1 на custom fields в Deal — Compliance-офицер видит структурированные данные, а не PDF на 40 страниц.

Аудитория. C-level или compliance / BD-руководитель со стороны B2B-партнёра. Среднее время заполнения: 35–50 минут при наличии документов на руках. Форма поддерживает save-and-resume — Zoho Survey генерирует уникальную resume-ссылку.

Языки. EN (основной) + RU (перевод) — multi-language feature Zoho Survey. Клиент выбирает язык на Page 0. Маппинг в Deal единый, независимо от языка заполнения.

Интеграции.

  • Zoho WorkDrive — upload-поля сохраняют файлы в папку /Deals/{Deal_ID}/Survey_Attachments/.
  • Zoho CRM — webhook on submit → Deluge listing_handle_survey_webhook().
  • Slack — уведомление менеджеру о сабмите + алёрт compliance-офицеру при обнаружении стоп-фактора (канал #listings / #compliance-listings).

2. Конвенции

3. Page 0 — Welcome & authentication

Вступительный экран · нет вопросов, только verification
PAGE 0 / 11

Zoho Survey ссылка на форму содержит pre-filled параметры: ?deal_id={Deal_ID}&token={survey_token}. Токен проверяется встроенным JavaScript (или server-side через webhook validation hook) — если deal_id/token не валиден, форма не отдаётся.

На экране — приветствие от имени Vault, ожидаемое время заполнения (35–50 мин), список документов, которые нужно подготовить заранее, контактный email для вопросов (partnerships@vault.ist), и выбор языка (EN / RU).

Pre-fill из Deal: Contact_Name, Contact_Email, Partner_Company — подставляются как read-only context на welcome-экране.

Server-side processing (listing_handle_survey_webhook pseudocode)

void listing_handle_survey_webhook(Map payload)
{
  deal_id = payload.get("deal_id");
  answers = payload.get("answers");
  attachments = payload.get("attachments");

  // 1. Map answers to Deal custom fields (dictionary-based)
  update_map = Map();
  FIELD_MAP = {
    "Q_identity_project_name": "Token_Project_Name",
    "Q_identity_ticker": "Token_Ticker",
    // ... full map as per pills in each question
  };
  for each qid in answers
  {
    crm_field = FIELD_MAP.get(qid);
    if (crm_field != null) update_map.put(crm_field, answers.get(qid));
  }

  // 2. Attachments: copy files to /Deals/{Deal_ID}/Survey_Attachments/ via WorkDrive API,
  //    write back the final WorkDrive URLs
  for each att_key in attachments
  {
    src = attachments.get(att_key);
    if (src != null)
    {
      final_url = listing_move_to_deal_folder(deal_id, src, att_key);
      crm_field = FIELD_MAP.get(att_key);
      update_map.put(crm_field, final_url);
    }
  }

  // 3. Stop-factor evaluation
  stop_reason = listing_evaluate_stop_factors(answers);
  if (stop_reason != null)
  {
    update_map.put("Listing_Stop_Reason", stop_reason);
    update_map.put("Stage", "Отказ / " + stop_reason);
  }
  else
  {
    update_map.put("Stage", "Compliance Review");
  }

  // 4. Mode detection
  mode = listing_detect_mode(answers.get("Q_token_network"), answers.get("Q_market_exchanges"));
  update_map.put("Listing_Mode_Candidate", mode);

  // 4b. v2: Service Tier pre-qualification
  // Basic всегда eligible (если не stop-factor).
  update_map.put("Listing_Tier_Basic_Eligible", stop_reason == null);
  // Swap: DEX TVL >= threshold OR whitelisted CEX-листинг.
  dex_tvl       = toLong(answers.getOrDefault("Q_market_dex_tvl_usd", 0));
  cex_listed_wl = listing_has_whitelisted_cex(answers.get("Q_market_exchanges"));
  update_map.put("Listing_Tier_Swap_Eligible",
                 dex_tvl >= SWAP_TVL_THRESHOLD || cex_listed_wl);
  // Utility-Card: требует Swap + USDT/USDC в accepted payment methods.
  pay_methods   = answers.getOrDefault("Q_commercial_accepted_pay", List());
  update_map.put("Listing_Tier_Utility_Card_Eligible",
                 update_map.get("Listing_Tier_Swap_Eligible") == true
                 && (pay_methods.contains("USDT") || pay_methods.contains("USDC")));
  // OnOff-Ramp pre-screen.
  tier_req = answers.getOrDefault("Q_tier_requested", List());
  if (tier_req.contains("OnOff-Ramp"))
  {
    buyback_ok = (answers.get("Q_onramp_buyback_ready") == "Yes");
    model_ok   = (update_map.get("Listing_Buyback_Model_Url") != null);
    ack_ok     = (answers.get("Q_onramp_ack_triumvirate") == true);
    update_map.put("Listing_Tier_OnRamp_PreScreen",
                   (buyback_ok && model_ok && ack_ok) ? "passed" : "failed");
  }
  else
  {
    update_map.put("Listing_Tier_OnRamp_PreScreen", "not_requested");
  }

  // 5. Update Deal
  zoho.crm.updateRecord("Deals", deal_id, update_map);

  // 6. Notify
  listing_slack_notify(deal_id, stop_reason, mode);
}
